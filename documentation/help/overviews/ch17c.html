Chapter 17c — DCC Detector Protocol

The DCC Detector Protocol reports track occupancy and locomotive detection using OpenLCB Event IDs. A detector node produces events that encode the detector's identity, the occupancy status (direction of travel), and the DCC address of the detected locomotive. This allows layout automation systems to track train positions on the network.

Source files: src/openlcb/openlcb_application_dcc_detector.c, src/openlcb/openlcb_application_dcc_detector.h
Transport: DCC Detector events use the standard Event Transport Protocol (Ch 13). The detector module provides encoding/decoding utilities for the Event ID format — it does not define new MTIs or message types.

17c.1 Event ID Bit-Field Layout

Each DCC detector Event ID packs three fields into 64 bits:

BitsWidthFieldDescription
[63:16]48 bitsDetector Node IDThe Node ID of the detector hardware (identifies the physical block)
[15:14]2 bitsDirection / OccupancyEncodes occupancy state and travel direction
[13:0]14 bitsDCC AddressThe address of the detected locomotive (encoding varies by type)
 
// Bit layout constants
#define DCC_DETECTOR_ADDRESS_BITS      14
#define DCC_DETECTOR_ADDRESS_MASK      0x3FFF
#define DCC_DETECTOR_DIRECTION_MASK    0xC000
#define DCC_DETECTOR_DIRECTION_SHIFT   14

17c.2 Direction / Occupancy Status

ValueConstantMeaning
0x00DCC_DETECTOR_UNOCCUPIEDBlock is unoccupied (exit event)
0x01DCC_DETECTOR_OCCUPIED_FORWARDOccupied, locomotive traveling forward
0x02DCC_DETECTOR_OCCUPIED_REVERSEOccupied, locomotive traveling in reverse
0x03DCC_DETECTOR_OCCUPIED_UNKNOWNOccupied, direction unknown

The direction bits allow automation systems to determine not just whether a block is occupied, but which direction the train is moving through the detection zone.

17c.3 Address Types

The 14-bit address field supports multiple DCC addressing modes, distinguished by prefix bits:

TypePrefixRangeDescription
Long addressNone0 – 0x37FF14-bit extended DCC address (excludes reserved ranges 0x3800-0x3FFF)
Short address0x38xx0 – 1277-bit basic DCC address (prefix DCC_DETECTOR_SHORT_ADDRESS_PREFIX)
Consist address0x39xx0 – 1277-bit consist address (prefix DCC_DETECTOR_CONSIST_ADDRESS_PREFIX)
Track empty0x3800N/ASentinel value indicating the track section is empty
Reserved range: Address values 0x3A00 through 0x3FFF are reserved and must not be used. The address prefixes 0x38 and 0x39 are allocated for short and consist addresses respectively.

17c.4 Encoding Functions

These functions build Event IDs and format raw address values:

FunctionPurpose
OpenLcbApplicationDccDetector_encode_event_id(detector_node_id, direction, raw_address_14) Build a complete 64-bit Event ID from detector Node ID, direction enum, and 14-bit raw address
OpenLcbApplicationDccDetector_make_short_address(short_address) Convert a 7-bit short address (0-127) to 14-bit format with 0x38 prefix
OpenLcbApplicationDccDetector_make_consist_address(consist_address) Convert a 7-bit consist address (0-127) to 14-bit format with 0x39 prefix

Example: Encoding a Detection Event

// Locomotive 42 (short address) detected moving forward in block 05.02.01.01.22.00
event_id_t event = OpenLcbApplicationDccDetector_encode_event_id(
    0x050201012200ULL,                                        // detector node ID
    DCC_DETECTOR_OCCUPIED_FORWARD,                            // direction
    OpenLcbApplicationDccDetector_make_short_address(42)      // 0x382A
);
// Result: 0x0502010122004000 | (0x382A) = 0x050201012200782A

17c.5 Decoding Functions

These functions extract fields from a received DCC detector Event ID:

FunctionReturnsPurpose
extract_direction(event_id) dcc_detector_direction_enum Occupancy status and travel direction
extract_address_type(event_id) dcc_detector_address_type_enum Whether address is long, short, consist, or track-empty
extract_raw_address(event_id) uint16_t 14-bit raw address including type prefix
extract_dcc_address(event_id) uint16_t Usable DCC address (prefix stripped for short/consist)
extract_detector_id(event_id) node_id_t 48-bit Node ID of the detector hardware
is_track_empty(event_id) bool True if the address field is the track-empty sentinel (0x3800)
