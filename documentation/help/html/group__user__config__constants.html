Detailed Description

Compile-time configuration parameters for buffer sizes and limits.

These constants control memory allocation for the OpenLCB library. They can be overridden by defining them in your compiler's preprocessor macros before including this header.

Buffer Pool Configuration: The total number of message buffers is the sum of all four buffer types. For 8-bit processors, this sum must not exceed 126 (signed 8-bit limit).

Typical Configurations:

  • Small nodes: BASIC=10, DATAGRAM=2, SNIP=2, STREAM=0, NODE=1
  • Medium nodes: BASIC=20, DATAGRAM=4, SNIP=4, STREAM=1, NODE=2
  • Large nodes: BASIC=32, DATAGRAM=4, SNIP=4, STREAM=1, NODE=4

Event Configuration: Producer/consumer counts determine how many events a node can handle. Maximum is 126 for 8-bit processors.

Note
Override these in your build system, not by editing this file
Test defaults are provided for Google Tests
