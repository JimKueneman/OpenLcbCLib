◆ on_search_no_match_with_allocate

openlcb_node_t *(* on_search_no_match_with_allocate) (event_id_t search_event_id)

Called when no train node matches after the 200ms allocate window.

Fires only for searches with TRAIN_SEARCH_FLAG_ALLOCATE set, and only after TRAIN_SEARCH_ALLOCATE_TIMEOUT_TICKS 100ms ticks have elapsed with no Producer Identified Set reply from any node on the network. The application must allocate a new virtual train node and call OpenLcbApplicationTrain_send_search_match to emit the Producer Identified reply.
