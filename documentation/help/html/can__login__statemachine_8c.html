◆ CanLoginStateMachine_run()

void CanLoginStateMachine_run ( can_statemachine_info_t * can_statemachine_info)

Runs the CAN login state machine.

Runs the CAN login state machine for one iteration.

Algorithm:

  1. Read node's current run_state
  2. Switch on run_state value
  3. For each state case:
    • Call appropriate state handler via interface
    • Return immediately after handler completes
  4. Default case returns without action

Use cases:

  • Called from main application loop for nodes in login sequence
  • Called repeatedly until node reaches permitted state
* @param can_statemachine_info Pointer to a structure that contains the node
* and output message buffer for the login sequence
*
Warning
Structure pointer must not be NULL
Node pointer within structure must not be NULL
Outgoing message buffer must be valid
NOT thread-safe
Attention
Modifies node run_state as it progresses through sequence
May set login_outgoing_can_msg_valid flag
Note
Call from the main application loop as fast as possible
State machine returns immediately after dispatching to handler
See also
CanLoginMessageHandler_state_init - First state handler
CanLoginMessageHandler_state_load_amd - Final state handler
