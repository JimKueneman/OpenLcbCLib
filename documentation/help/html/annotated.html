Data Structures
Here are the data structures with brief descriptions:
 Calias_mapping_info_structStructure to hold an array of alias_mapping_t structures and a flag to flag the main loop that at least one of the containing mappings has detected a duplicate Alias ID If during a message/frame reception it is found that some other node is using an Alias that is associated with one of our nodes the the has_duplicate_alias flag is set. This mean at least one mapping in the array is a duplicate
 Calias_mapping_structStructure to hold a shadow buffer of Node ID/Alias pairs for Nodes that have been allocated and logged into the network. If during a message/frame reception it is found that some other node is using an Alias that is associated with one of our nodes the is_duplicate flag is set. Likewise once the node has become Permitted on the network the is_permitted flag is set. This allows the interrupt or thread on receiving the frame can use this structure to make these checks/flags and the main loop can access this structure (using the lock/unlock calls) to safely peek in and see if something needs to be handled in the main loops context
